นี่คือบางส่วนที่พบมากที่สุด


1xx ข้อมูลตอบกลับ: ใช้เพื่อระบุว่าได้รับคำขอแล้วและกำลังดำเนินการต่อไป

2xx คำตอบสำเร็จ: ใช้เพื่อระบุว่าคำขอได้รับ เข้าใจ และยอมรับสำเร็จแล้ว

3xx Redirection Responses: ใช้เพื่อระบุว่า User Agent (ซึ่งโดยทั่วไปคือเว็บเบราว์เซอร์) จำเป็นต้องดำเนินการเพิ่มเติมเพื่อให้ดำเนินการตามคำขอ

4xx Client Error Responses: ใช้เพื่อระบุว่ามีข้อผิดพลาดกับคำขอ โดยทั่วไปเกิดจากข้อมูลที่ไคลเอ็นต์ให้มาไม่ถูกต้องหรือไม่สมบูรณ์

	400 คำขอไม่ถูกต้อง: คำขอมีรูปแบบไม่ถูกต้องหรือไม่สมบูรณ์ และเซิร์ฟเวอร์ไม่สามารถเข้าใจได้ (โดยปกติเกิดจากปัญหาเกี่ยวกับคำขอของคุณ)
	401 ไม่ได้รับอนุญาต: จำเป็นต้องมีการรับรองความถูกต้อง และผู้ใช้ไม่ได้ให้ข้อมูลประจำตัวที่ถูกต้อง
	403 ห้าม: เซิร์ฟเวอร์ปฏิเสธที่จะปฏิบัติตามคำขอ โดยปกติแล้วเป็นเพราะผู้ใช้ไม่ได้รับอนุญาตให้เข้าถึงทรัพยากรที่ร้องขอ (เมื่อคุณไม่ได้รับอนุญาตให้เข้าถึงเนื้อหาที่ร้องขอ)
	404 ไม่พบ: เซิร์ฟเวอร์ไม่พบทรัพยากรหรือเพจที่คุณกำลังมองหา
	ไม่อนุญาตให้ใช้เมธอด 405: เมธอด HTTP ที่ใช้ในคำขอไม่ได้รับการสนับสนุนสำหรับทรัพยากรที่ร้องขอ
	408 Request Timeout: เซิร์ฟเวอร์หมดเวลารอคำขอจากลูกค้า

5xx Server Error Responses: ใช้เพื่อระบุว่ามีข้อผิดพลาดในฝั่งเซิร์ฟเวอร์

	ข้อผิดพลาดเซิร์ฟเวอร์ภายใน 500: เซิร์ฟเวอร์พบข้อผิดพลาดขณะประมวลผลคำขอและไม่สามารถดำเนินการให้เสร็จสิ้นได้
	501 ไม่ได้ใช้งาน: เซิร์ฟเวอร์ไม่รองรับวิธีการที่ร้องขอ
	502 Bad Gateway: เซิร์ฟเวอร์ที่ทำหน้าที่เป็นเกตเวย์หรือพร็อกซีได้รับการตอบสนองที่ไม่ถูกต้องจากเซิร์ฟเวอร์อัพสตรีม
	บริการ 503 ไม่พร้อมใช้งาน: ขณะนี้เซิร์ฟเวอร์ไม่สามารถจัดการคำขอได้เนื่องจากการโหลดเกินชั่วคราวหรือการบำรุงรักษา (เมื่อเซิร์ฟเวอร์ไม่สามารถจัดการคำขอของคุณได้ชั่วคราว)
	หมดเวลาเกตเวย์ 504: เซิร์ฟเวอร์ที่ทำหน้าที่เป็นเกตเวย์หรือพร็อกซีไม่ได้รับการตอบสนองจากเซิร์ฟเวอร์อัปสตรีมภายในระยะเวลาที่กำหนด
